# Strategic Analysis Lab

AI-assisted analysis of complex strategic questions using web evidence and structured reasoning.

## Overview

Strategic Analysis Lab is a lightweight research tool that analyzes complex strategic questions using large language models and real public web data.

The system retrieves factual snippets from the web, injects them into the model prompt, and produces a structured analysis grounded in cited sources.

This project is designed as a sandbox for exploring AI-assisted reasoning about large-scale systems such as markets, infrastructure, operational networks, and technology adoption.

The current version focuses on reasoning quality rather than deterministic simulation.

## Key Features

- Web-grounded reasoning using public data
- Structured strategic analysis output
- Retrieval-augmented language model prompting
- Built-in API safety controls (rate limits and token caps)
- Simple architecture designed for experimentation and extension

## Example Questions

Examples of questions the system can analyze:

- Can a new technology reach mass adoption within a decade?
- What infrastructure would be required to support large-scale service networks?
- What market size is required to justify a large capital investment?
- What operational constraints limit system scaling?

## How It Works

Strategic Analysis Lab follows a simple retrieval-augmented reasoning pipeline:

```
User Question
→ Web Retrieval (search provider)
→ Snippet Extraction
→ Prompt Construction
→ Language Model Reasoning
→ Structured Analysis Output
```

The system retrieves a small number of factual snippets from the web and injects them into the prompt so the model can reason using real-world evidence.

This helps reduce hallucinations and grounds the analysis in verifiable sources.

## System Components

The project contains four main modules.

### Question Interface

The user provides a strategic question through a simple web interface.

Examples include questions about market scale, infrastructure requirements, or operational constraints.

### Retrieval Layer

The retrieval layer queries a web search provider such as Tavily to obtain relevant public information.

The system extracts a small number of snippets from trusted domains and passes them to the reasoning engine.

### Reasoning Engine

The reasoning engine sends the question and snippets to a language model provided by OpenAI.

The model produces a structured analysis using the retrieved evidence.

Typical output sections include:

- Interpretation
- Executive answer
- Assumptions
- System framing
- Back-of-envelope math
- Sensitivity drivers
- Sanity checks using public data

### Safety Controls

To prevent uncontrolled API usage the system includes several protections:

- Rate limiting per IP address
- Maximum token limits for model responses
- Restricted model allowlist
- Server-side kill switch via environment variable

These controls make the application safe to run publicly.

## What the System Does Not Do

This version does not perform deterministic simulation or forecasting.

It does not:

- run simulations
- build mathematical optimization models
- forecast using statistical models
- compute system dynamics

All reasoning is generated by the language model using retrieved web evidence.

Future versions may introduce deterministic computation layers.

## Architecture

The system pipeline is intentionally simple:

```
User Question
→ Web Retrieval
→ Snippet Selection
→ Prompt Construction
→ Language Model Analysis
→ Structured Strategic Output
```

The language model does not access the internet directly.
All external information is retrieved by the system and passed into the prompt.

## Local Setup

Clone the repository:

```bash
git clone https://github.com/rafaelmaranon/strategic-analysis-lab.git
cd strategic-analysis-lab
```

Install dependencies:

```bash
npm install
```

Create a `.env.local` file in the root directory:

```
OPENAI_API_KEY=your_openai_key
TAVILY_API_KEY=your_search_key
UPSTASH_REDIS_REST_URL=your_redis_url
UPSTASH_REDIS_REST_TOKEN=your_redis_token
API_DISABLED=false
```

Run the development server:

```bash
npm run dev
```

Open the application in the browser:

```
http://localhost:3000/examples-lab
```

## Deployment

The project can be deployed using Vercel.

### Steps

1. Push the repository to GitHub
2. Import the repository into Vercel
3. Add the required environment variables in the Vercel project settings
4. Deploy the project

After deployment the application will be available at the Vercel project URL.

## Notes

Strategic Analysis Lab is intended for experimentation with AI-assisted reasoning and retrieval-augmented analysis.

Results depend on the quality of retrieved sources and should not be considered authoritative or financial advice.
